# Темпоральное расширение доменной теории балансов

## Преамбула

Данный документ является расширением формальной теории доменов, правил, материализаций и текущих балансов.

Базовая теория (balance-domains) определяет:

- пространство измерений $\mathbb D$;
- транзакции с числовым вкладом;
- аддитивные домены;
- правила (scope) и материализации (groupBy);
- семейства балансов $F=(D,R,M)$;
- операции conversion, roll-up, derived composition.

Базовая теория работает с текущими балансами: значение balance row — это накопленная сумма всех подходящих транзакций без ограничений по времени.

Темпоральное расширение добавляет время как специальное измерение, позволяя вести:

- обороты по периодам (turnover);
- кумулятивные балансы на дату (balance as of date);
- иерархическую периодизацию (day → month → quarter → year).

При этом базовое ядро — домены, правила, материализации, аддитивность — не изменяется.

---

## T1. Временна́я метка транзакции

Каждая транзакция $e$ несёт временну́ю метку

$$\tau(e) \in \mathbb T,$$

где $\mathbb T$ — линейно упорядоченное множество моментов времени (например, $\mathbb T = \mathbb R$ или $\mathbb T = \mathbb Z$ для дискретных дней).

### Бизнес-смысл

$\tau(e)$ — это момент, когда транзакция произошла или была зафиксирована в системе.

Например:

- $\tau(e) = \text{2025-03-15T14:30:00}$

Временна́я метка не является обычным измерением из $\mathbb D$. Она не участвует в доменном префиксе, не входит в scope правила и не включается в groupBy напрямую. Вместо этого время обрабатывается через специальный механизм периодизации.

### Почему время — не обычное измерение

Обычные измерения из $\mathbb D$ обладают следующими свойствами:

1. конечное множество значений $Val(d)$;
2. значения не упорядочены (или порядок не существенен для аддитивности);
3. groupBy по измерению порождает конечное фиксированное множество ключей.

Время отличается:

1. множество значений бесконечно и монотонно растёт;
2. значения строго упорядочены, и порядок существенен;
3. одна и та же метка может быть агрегирована на разных уровнях гранулярности (day, month, year);
4. существует кумулятивная семантика (balance as of), которая не выражается простым groupBy.

Поэтому время вводится как отдельная структура, а не как элемент $\mathbb D$.

## T2. Гранулярность и функция периодизации

### T2.1. Гранулярность

Гранулярность $g$ — это способ отображения точного момента времени в дискретный период.

Пусть задано конечное множество гранулярностей

$$\mathbb G = \{g_1, \dots, g_s\},$$

упорядоченное от самой грубой к самой детальной:

$$g_1 \prec g_2 \prec \dots \prec g_s.$$

### T2.2. Функция периодизации

Для каждой гранулярности $g \in \mathbb G$ задана функция периодизации

$$\pi_g : \mathbb T \to Per(g),$$

где $Per(g)$ — множество периодов гранулярности $g$.

### T2.3. Согласованность гранулярностей

Если $g_a \prec g_b$ (то есть $g_a$ грубее $g_b$), то существует функция свёртки

$$\rho_{b \to a} : Per(g_b) \to Per(g_a),$$

такая что для любого момента $t \in \mathbb T$:

$$\pi_{g_a}(t) = \rho_{b \to a}(\pi_{g_b}(t)).$$

Иными словами, более детальный период однозначно определяет более грубый.

### Бизнес-смысл

Типичный набор гранулярностей:

$$\mathbb G = \{year, quarter, month, week, day\}.$$

Функции периодизации:

- $\pi_{day}(\text{2025-03-15}) = \text{2025-03-15}$
- $\pi_{month}(\text{2025-03-15}) = \text{2025-03}$
- $\pi_{quarter}(\text{2025-03-15}) = \text{2025-Q1}$
- $\pi_{year}(\text{2025-03-15}) = \text{2025}$

Свёртка:

- $\rho_{day \to month}(\text{2025-03-15}) = \text{2025-03}$
- $\rho_{month \to year}(\text{2025-03}) = \text{2025}$

Это означает, что зная день, можно однозначно определить месяц, квартал и год.

## T3. Темпоральная материализация

### T3.1. Расширение семейства балансов

В базовой теории семейство определяется как

$$F = (D, R, M).$$

В темпоральном расширении семейство определяется как

$$F = (D, R, M, \Theta),$$

где $\Theta$ — темпоральная спецификация.

### T3.2. Темпоральная спецификация

Темпоральная спецификация $\Theta$ принимает одно из трёх значений:

1. **Accumulated** ($\Theta = acc$): текущий баланс, без разбиения по времени.
2. **Turnover** ($\Theta = turn(g)$): оборот по периодам гранулярности $g$.
3. **As-of** ($\Theta = asof(g)$): кумулятивный баланс на конец каждого периода гранулярности $g$.

### T3.3. Совместимость с базовой теорией

Если $\Theta = acc$, то семейство $F = (D, R, M, acc)$ полностью эквивалентно семейству базовой теории $F = (D, R, M)$.

Это означает, что базовая теория является частным случаем темпорального расширения.

## T4. Расширенный канонический путь

### T4.1. Определение

Для семейства $F = (D, R, M, \Theta)$ и транзакции $e$ расширенный канонический путь определяется как:

$$BalPath_F(e) = Canon\bigl(DomPath(D),\ Scope_R,\ Key_M(e),\ TemporalKey_\Theta(e)\bigr),$$

где $TemporalKey_\Theta(e)$ определяется следующим образом:

- если $\Theta = acc$: $TemporalKey_\Theta(e) = \varepsilon$ (пусто, нет темпоральной компоненты);
- если $\Theta = turn(g)$: $TemporalKey_\Theta(e) = \pi_g(\tau(e))$;
- если $\Theta = asof(g)$: $TemporalKey_\Theta(e)$ определяется отдельно (см. §T6).

### T4.2. Бизнес-смысл

Для семейства с $\Theta = turn(month)$ и groupBy = (country, vertical) пути будут:

- `trx / EUR / country=EE / vertical=Food / month=2025-03`
- `trx / EUR / country=EE / vertical=Food / month=2025-04`
- `trx / EUR / country=LV / vertical=Grocery / month=2025-03`

Для семейства с $\Theta = acc$ пути остаются как в базовой теории:

- `trx / EUR / country=EE / vertical=Food`

## T5. Turnover balance

### T5.1. Определение

Для семейства $F = (D, R, M, turn(g))$ баланс по расширенному пути $p$ определяется как

$$B_F(p) = \sum_{\substack{e \in E,\ e \in D,\ Scope_R(e)=1,\\ Key_M(e) = k_M,\ \pi_g(\tau(e)) = \omega}} Val(e),$$

где $p = Canon(DomPath(D), Scope_R, k_M, \omega)$.

### T5.2. Объяснение

Turnover — это сумма вкладов транзакций, которые:

- принадлежат домену;
- удовлетворяют scope правила;
- имеют одинаковый ключ материализации;
- **и попадают в один и тот же период.**

### T5.3. Аддитивность turnover

Turnover balance аддитивен по транзакциям:

$$B_F^{E_1 \cup E_2}(p) = B_F^{E_1}(p) + B_F^{E_2}(p).$$

Это прямое следствие определения: формула суммирования одинакова, просто множество транзакций расширяется.

### T5.4. Бизнес-смысл

Если семейство — это «GMV по стране и месяцу», то:

- $B(\text{EE}, \text{Food}, \text{2025-03}) = 15\,000$ означает: за март 2025 GMV по Эстонии/Food составил 15 000.
- $B(\text{EE}, \text{Food}, \text{2025-04}) = 18\,000$ означает: за апрель 2025 GMV по Эстонии/Food составил 18 000.

Это именно обороты — суммы за период, а не накопленные значения.

---

## T6. As-of balance (кумулятивный баланс)

### T6.1. Интуиция

As-of balance отвечает на вопрос: «какой баланс на конец данного периода?»

Это не сумма за период, а сумма всех транзакций от начала времён до конца данного периода.

### T6.2. Определение

Для семейства $F = (D, R, M, asof(g))$ и периода $\omega \in Per(g)$ кумулятивный баланс определяется как

$$B_F^{asof}(k_M, \omega) = \sum_{\substack{e \in E,\ e \in D,\ Scope_R(e)=1,\\ Key_M(e) = k_M,\ \pi_g(\tau(e)) \le \omega}} Val(e).$$

### T6.3. Связь с turnover

As-of balance выражается через turnover:

$$B_F^{asof}(k_M, \omega) = \sum_{\omega' \le \omega} B_F^{turn}(k_M, \omega'),$$

где суммирование идёт по всем периодам $\omega'$ гранулярности $g$, предшествующим $\omega$ включительно.

Обратно, turnover выражается через as-of:

$$B_F^{turn}(k_M, \omega) = B_F^{asof}(k_M, \omega) - B_F^{asof}(k_M, \omega^{-}),$$

где $\omega^{-}$ — предыдущий период.

### T6.4. Аддитивность as-of

As-of balance аддитивен по транзакциям:

$$B_F^{asof,\, E_1 \cup E_2}(k_M, \omega) = B_F^{asof,\, E_1}(k_M, \omega) + B_F^{asof,\, E_2}(k_M, \omega).$$

Однако as-of balance **не аддитивен по периодам**: нельзя сложить as-of за март и as-of за апрель — получится удвоенная масса.

### T6.5. Бизнес-смысл

Если ведётся бухгалтерский баланс по счёту:

- $B^{asof}(\text{expense/ops\_cost/EE}, \text{2025-03}) = 450\,000$ означает: на конец марта 2025 накопленные расходы ops_cost по Эстонии составили 450 000.
- $B^{asof}(\text{expense/ops\_cost/EE}, \text{2025-04}) = 570\,000$ означает: на конец апреля — 570 000.
- Оборот за апрель: $570\,000 - 450\,000 = 120\,000$.

## T7. Temporal roll-up

### T7.1. Определение

Temporal roll-up — это частный случай conversion между семействами с разной гранулярностью.

Пусть есть два семейства:

- $F_b = (D, R, M, turn(g_b))$ — детальное (например, daily);
- $F_a = (D, R, M, turn(g_a))$ — грубое (например, monthly).

Если $g_a \prec g_b$, то:

$$B_{F_a}^{turn}(k_M, \omega_a) = \sum_{\substack{\omega_b \in Per(g_b),\\ \rho_{b \to a}(\omega_b) = \omega_a}} B_{F_b}^{turn}(k_M, \omega_b).$$

### T7.2. Бизнес-смысл

Месячный GMV — это сумма дневных GMV за все дни этого месяца.

$$GMV(\text{EE}, \text{Food}, \text{2025-03}) = \sum_{d \in days(\text{2025-03})} GMV(\text{EE}, \text{Food}, d).$$

Это обычный roll-up из базовой теории, применённый к темпоральному измерению.

### T7.3. Roll-up для as-of

Для as-of balance roll-up между гранулярностями определяется иначе.

Если $g_a \prec g_b$ (например, $g_a = month$, $g_b = day$), то:

$$B_{F_a}^{asof}(k_M, \omega_a) = B_{F_b}^{asof}(k_M, \text{last}(\omega_a)),$$

где $\text{last}(\omega_a)$ — последний период $g_b$, попадающий в $\omega_a$.

### Бизнес-смысл

As-of баланс на конец месяца равен as-of балансу на последний день этого месяца.

$$B^{asof}(\text{EE}, \text{2025-03}) = B^{asof}(\text{EE}, \text{2025-03-31}).$$

Это не сумма дневных as-of значений (это было бы бессмысленно), а значение на конец последнего дня.

## T8. Связь accumulated, turnover и as-of

### T8.1. Accumulated как предел as-of

Accumulated balance из базовой теории — это as-of balance с бесконечным горизонтом:

$$B_F^{acc}(k_M) = B_F^{asof}(k_M, +\infty) = \sum_{\omega \in Per(g)} B_F^{turn}(k_M, \omega).$$

### T8.2. Единая картина

Три типа темпоральной спецификации образуют следующую иерархию:

1. **Turnover** $B^{turn}(k_M, \omega)$ — базовый объект: сумма за один период.
2. **As-of** $B^{asof}(k_M, \omega) = \sum_{\omega' \le \omega} B^{turn}(k_M, \omega')$ — кумулятивная сумма turnover.
3. **Accumulated** $B^{acc}(k_M) = \sum_{\omega} B^{turn}(k_M, \omega)$ — полная сумма turnover по всем периодам.

Таким образом, turnover — это фундаментальный темпоральный объект, а as-of и accumulated — производные.

## T9. Темпоральные операции и базовые операции

### T9.1. Conversion

Conversion из базовой теории работает без изменений.

Если задан conversion $F_A \to F_B$ c отображением путей $\Psi$ и числовым преобразованием $\Gamma$, то для темпоральных семейств conversion применяется поточечно по периодам:

$$Conv_{F_A \to F_B}^{turn}(p_B, \omega) = \sum_{\substack{p_A \in PathSpace(F_A),\\ \Psi(p_A) = p_B}} \Gamma_{p_A \to p_B}\bigl(B_{F_A}^{turn}(p_A, \omega)\bigr).$$

### T9.2. Derived composition

Derived composition также применяется поточечно:

$$B_{F_*}^{turn}(p, \omega) = \Phi\bigl(B_{F_1}^{turn}(p, \omega), \dots, B_{F_n}^{turn}(p, \omega)\bigr).$$

### T9.3. Бизнес-смысл

Месячная прибыль:

$$Profit^{turn}(\text{EE}, \text{2025-03}) = Income^{turn}(\text{EE}, \text{2025-03}) - Expense^{turn}(\text{EE}, \text{2025-03}).$$

Кумулятивная прибыль на конец марта:

$$Profit^{asof}(\text{EE}, \text{2025-03}) = Income^{asof}(\text{EE}, \text{2025-03}) - Expense^{asof}(\text{EE}, \text{2025-03}).$$

Обе формулы корректны и следуют из поточечного применения derived composition.

## T10. Path-space-disjointness с учётом темпоральности

### T10.1. Расширенное правило

В базовой теории требуется, чтобы разные семейства внутри одного домена не порождали пересекающиеся пространства путей.

В темпоральном расширении путь включает темпоральную компоненту, поэтому:

- семейство с $\Theta = turn(month)$ и семейство с $\Theta = turn(day)$ порождают разные пространства путей (пути отличаются гранулярностью);
- семейство с $\Theta = acc$ и семейство с $\Theta = turn(month)$ также порождают разные пространства путей (у первого нет темпоральной компоненты).

### T10.2. Формально

Два семейства $F_1 = (D, R_1, M_1, \Theta_1)$ и $F_2 = (D, R_2, M_2, \Theta_2)$ имеют непересекающиеся пространства путей, если:

$$PathSpace(F_1) \cap PathSpace(F_2) = \varnothing.$$

Достаточное условие: если $\Theta_1 \ne \Theta_2$, то пути автоматически различаются по темпоральной компоненте.

Если $\Theta_1 = \Theta_2$, применяется обычное правило path-space-disjointness из базовой теории.

## T11. Материализация и хранение

### T11.1. Turnover как базовый объект хранения

Из трёх темпоральных объектов turnover является фундаментальным:

- as-of вычисляется как кумулятивная сумма turnover;
- accumulated вычисляется как полная сумма turnover.

Поэтому достаточно материализовать turnover, а as-of и accumulated вычислять (или кэшировать как производные).

### T11.2. Инкрементальность записи

При поступлении новой транзакции $e$:

1. определяется домен $D$;
2. для каждого активного семейства $F = (D, R, M, \Theta)$:
   - проверяется $Scope_R(e) = 1$;
   - вычисляется $Key_M(e)$;
   - если $\Theta = turn(g)$: вычисляется $\omega = \pi_g(\tau(e))$;
   - обновляется $B_F(k_M, \omega)\ {+}{=}\ Val(e)$.
3. если семейство с $\Theta = acc$: обновляется $B_F(k_M)\ {+}{=}\ Val(e)$.

Сложность записи: $O(k)$, где $k$ — число активных семейств в домене.

### T11.3. Сложность чтения

- Turnover по ключу $(k_M, \omega)$: $O(1)$.
- Accumulated по ключу $k_M$: $O(1)$ (если материализован).
- As-of по ключу $(k_M, \omega)$: $O(|\omega|)$ если вычисляется из turnover, или $O(1)$ если кэширован.

## T12. Ограничения темпорального расширения

Темпоральное расширение сознательно не включает:

1. **Bi-temporal модель** (transaction time vs. business time). Рассматривается только один timestamp на транзакцию.
2. **Ретроактивные коррекции**. Если транзакция задним числом изменяет прошлый период, это не описывается как отдельная операция — это обычная транзакция с соответствующим $\tau(e)$.
3. **Snapshots**. Периодические фиксации состояния системы — это инфраструктурная операция, а не часть формальной теории.
4. **Time-weighted balances**. Средневзвешенные по времени значения (например, средний баланс за период) требуют дополнительного аппарата и не включены в текущее расширение.

Каждое из этих направлений может быть добавлено как последующее расширение, не изменяя текущее ядро.

---

## T13. Итоговая схема с темпоральностью

$$\text{Транзакция } (e, \tau(e)) \to \text{Домен } D \to \text{Правило } R \to \text{Материализация } M \to \text{Темпоральная спецификация } \Theta \to \text{Balance row}$$

где:

- домен изолирует аддитивность;
- правило фильтрует по scope;
- материализация определяет ключ;
- темпоральная спецификация определяет, как время участвует в ключе;
- balance row хранит результирующее значение.

---

## T14. Совместимость с базовым ядром

Темпоральное расширение не изменяет базовое ядро:

| Конструкция базовой теории | Изменение в расширении |
|---|---|
| Домен | Без изменений |
| Непересекаемость доменов | Без изменений |
| Правило (scope) | Без изменений |
| Материализация (groupBy) | Без изменений |
| Семейство $F = (D,R,M)$ | Расширено до $F = (D,R,M,\Theta)$ |
| Канонический путь | Расширен темпоральной компонентой |
| Аддитивность по транзакциям | Сохраняется |
| Path-space-disjointness | Расширено с учётом $\Theta$ |
| Conversion | Применяется поточечно по периодам |
| Derived composition | Применяется поточечно по периодам |
| Roll-up | Расширен: spatial и temporal |

Базовая теория является частным случаем темпорального расширения при $\Theta = acc$.

### T15.6. Пример: несколько семейств с разной темпоральной спецификацией в одном домене

Домен:

- `basis = trx`, `currency = EUR`

В одном домене одновременно существуют три семейства для одного и того же scope (`account_type = expense`, `groupBy = (country)`):

| семейство | $\Theta$ | что хранит |
|---|---|---|
| $F_1$ | $acc$ | текущий accumulated balance |
| $F_2$ | $turn(month)$ | месячные обороты |
| $F_3$ | $turn(day)$ | дневные обороты |

При поступлении транзакции $e$ с $\tau(e) = \text{2025-03-15}$, $Val(e) = 1\,000$, $country = \text{EE}$:

- $F_1$: обновляется $B_{F_1}(\text{EE})\ {+}{=}\ 1\,000$
- $F_2$: обновляется $B_{F_2}(\text{EE}, \text{2025-03})\ {+}{=}\ 1\,000$
- $F_3$: обновляется $B_{F_3}(\text{EE}, \text{2025-03-15})\ {+}{=}\ 1\,000$

Path-space-disjointness выполняется автоматически: пути $F_1$, $F_2$, $F_3$ различаются наличием и гранулярностью темпоральной компоненты.

Инварианты, которые должны выполняться:

$$B_{F_1}(\text{EE}) = \sum_{\omega} B_{F_2}(\text{EE}, \omega) = \sum_{d} B_{F_3}(\text{EE}, d)$$

$$B_{F_2}(\text{EE}, \text{2025-03}) = \sum_{d \in days(\text{2025-03})} B_{F_3}(\text{EE}, d)$$
